HTML

  • HTML provides the foundational structure of a webpage.
  • The head element contains information about the webpage, such as the character set and default view.
  • The body element represents the visible content shown to the user, such as presented text and images.
  • Semantic HTML tags are vital - they provide the meaning of the content contained and how they should be sectioned on the page.

CSS

  • CSS - cascading style sheet - defines how content is presented, i.e. aesthetics and formatting.
  • Inline CSS styles a particular page element through specific HTML tags.
    • This is useful for quick and permanent changes but less flexible as it is changing individual elements.
  • Internal Style Sheets contain CSS rules inside the head section.
    • Changes here apply to all tagged elements on the same page.
  • External Style Sheets are preferred and most common, where a separate file is linked in the head section.
    • This is emblematic of "separation of concerns" [SoC], or abstracting the program into distinct sections which each address a separate concern or detail.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: commit all files in working branch to stage, -A denotes all
  • git commit -m "put message here": commits to git repo with a message, keep it short and descriptive

Javascript

  • JS provides interactivity - clicking buttons, filling out a form, providing some sort of feedback to the user's input.
  • JS is linked to the HTML file in the body under the footer, as opposed to in the head for the CSS
  • Control Flow: Order matters so keep operations in mind - i.e. the DevTools console can't log variables before the variables are logged.
